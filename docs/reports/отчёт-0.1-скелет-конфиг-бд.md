## 1. Что требовалось

Поднять рабочий каркас приложения: структура пакетов, конфигурация, движок и сессии
SQLModel поверх SQLite. После 0.1 приложение **стартует, отдаёт `/health` и при первом
запуске создаёт файл БД**. Никакой доменной логики, никаких таблиц — чистая инфраструктура,
которую можно прогнать тестом и пощупать в браузере.

---

## 2. Решения по реализации

1. **Конфиг на `pydantic-settings`** (не голые константы): валидация типов и переопределение
   через env/`.env` «бесплатно», Pydantic и так в стеке. Префикс переменных — `STORE_`.
2. **Изоляция тестовой БД через двойной `monkeypatch`** (а не только `dependency_overrides`):
   DI-override не перехватывает `engine` внутри `init_db()`. Поэтому в фикстуре подменяются
   и `settings.db_path` → `tmp_path`, и модульный `app.database.engine` → временный движок.
   Боевой `data/store.db` тесты не трогают.
3. **`journal_mode=WAL`** добавлен как мягкая страховка при конкуренции потоков
   (окно pywebview + сервер uvicorn ходят в БД из разных потоков).
4. **pywebview в `run.py` уже сейчас** — валидирует весь стек одним запуском; при этом
   жёсткие критерии висят на uvicorn+TestClient и не зависят от GUI-окружения.

## 3. Что создано

| Файл | Назначение |
|---|---|
| `app/config.py` | `Settings(BaseSettings)`: `app_name`, `debug`, `db_path`, `db_echo`, `host`, `port`, свойство `database_url`; префикс `STORE_`, поддержка `.env`. Единый экземпляр `settings`. |
| `app/database.py` | Один `engine` (`check_same_thread=False`, `echo=db_echo`); listener `connect` с `PRAGMA foreign_keys=ON` и `journal_mode=WAL`; `get_session()` — зависимость FastAPI; `init_db()` — `mkdir(parents=True)` + `create_all`. Комментарий-маяк для импорта моделей в 0.2. |
| `app/main.py` | Фабрика `create_app()`, `lifespan` → `init_db()` на старте, `GET /health` → `{"status":"ok"}`, экспорт `app`. |
| `run.py` | Боевой запуск: uvicorn в daemon-потоке + окно pywebview на `http://host:port`. |
| `app/{models,routes,services,hardware}/__init__.py` | Пустые пакеты под будущие этапы. |
| `app/templates/.gitkeep`, `app/static/.gitkeep`, `data/.gitkeep` | Каталоги под будущее; `data/.gitkeep` остаётся в репозитории, сама БД — нет. |
| `requirements.txt` | 7 пакетов: fastapi, uvicorn[standard], sqlmodel, pydantic-settings, pywebview, pytest, httpx. |
| `.gitignore` | `__pycache__/`, `*.pyc`, `.venv/`, `.env`, `data/*.db` + `-wal`/`-shm`, `.pytest_cache/`. |
| `README.md` | Установка / запуск / отладка / тесты — 4 команды. |
| `tests/conftest.py` | Фикстуры `test_engine` (temp-БД + те же прагмы), `session`, `client`. |
| `tests/test_smoke.py` | `GET /health` → 200 и тело `{"status":"ok"}`. |
| `tests/test_db.py` | `init_db()` создаёт файл БД; `SELECT 1` живой; `PRAGMA foreign_keys = 1`. |

## 6. Соблюдение правил CLAUDE.md

- Остаёмся в Python; UI-фреймворков нет.
- Сырой SQL — только инфраструктурный (pragma на движке, `SELECT 1`/`text()` в тестах);
  доменной логики на сыром SQL нет.
- Специфика SQLite заперта в `database.py` (`check_same_thread`, прагмы) — переезд на
  Postgres сводится к смене `database_url`.
- Денежных/количественных полей в 0.1 нет; правило «деньги — `int`, количество — `Decimal`»
  держим для 0.2.
- Код, имена, коммиты — английский.
