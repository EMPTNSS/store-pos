> Roadmap: этап 3.3 — динамика цены закупки и продажи в виде графика / кривой (разд. 5.6),
> завершает этап 3 (карточка товара).

---

## 1. Что требовалось

Наполнить последнюю заглушку карточки (оставлена 3.1/3.2): секция **«Динамика цен»** с графиком,
показывающим, как менялись закупочная и розничная цена за время жизни товара. Этап только **читает**
уже копящуюся `price_history` (точка при создании товара 0.3 + при каждой правке цены 3.1),
хранилище не меняет.

---

## 2. Зафиксированные решения (согласованы до реализации)

1. **Серверный inline-SVG, без JS-библиотек.** График строится как `<svg>` прямо в Jinja-partial;
   ноль нового JavaScript, ноль CDN — в русле стека (весь UI — HTMX + серверный HTML). Геометрия —
   чистая функция в сервисном слое, тестируемая без веба.
2. **Ступенчатая кривая (step).** Цена держится горизонтально между правками, в момент изменения —
   вертикальный скачок. Честно отражает дискретную природу изменений цены.
3. **Один график, две кривые.** Закупка и продажа на общих осях, единый масштаб Y (зазор-наценка
   правдив), различаются цветом, с легендой. Ось X — реальное время точек.

---

## 3. Ключевые решения реализации

1. **Разделение «чтение ↔ геометрия».** `price_history(session, id)` тянет ряд из БД (хронологически);
   `build_price_chart(points, …)` — **чистая функция без session**, нормализует точки в координаты SVG.
   Шаблон отрисовывает готовые строки `points` и координаты маркеров — в шаблоне нет вычислений.
2. **Единый масштаб Y на обе кривые.** Диапазон `[min, max]` берётся по всем значениям (закупка +
   продажа), максимальная цена ложится на верх, минимальная — на низ. Зазор между кривыми = наценка
   в реальном масштабе.
3. **Ступень через промежуточную вершину.** Между точками i и i+1 добавляется `(x[i+1], y[i])`:
   горизонталь до момента изменения, затем вертикаль к новому значению. Последний сегмент дотянут до
   правого края.
4. **Деньги остаются `int` в копейках.** Во `float` уходят только пиксельные координаты SVG (это не
   деньги, округление некритично, CLAUDE.md правило 1 не нарушается). Подписи цен — фильтр `money`
   уже в шаблоне (int → «60.00 ₽»).
5. **Вырожденные случаи без деления на ноль.** Одна точка / один день (`t_max == t_min`) → `single`,
   плоская линия во всю ширину + подпись «Цена не менялась». Неизменная цена на разных датах
   (`y_max == y_min`) → диапазон Y паддится (±10 %, но не меньше 1 ₽), линия идёт по центру.
6. **Ленивый контейнер перезагружается сам.** Как и история движений (3.2): карточка перерисовывается
   `outerHTML` при сохранении паспорта (в т.ч. правка цены) → `hx-trigger="load"` срабатывает заново,
   график освежается свежей ступенью без ручного обновления.

## 7. Критерии приёмки (выполнены)

`pytest` — **205 passed** (9 новых в `test_price_chart.py`, регрессия 0–3.2 цела, было 196):

- **Чтение ряда:** `price_history` — точки по возрастанию `datetime`; после создания — одна точка,
  после правки цены — две в хронологическом порядке.
- **Ступень:** для двух точек `buy_line` содержит промежуточную вершину `(x1, y0)` (горизонталь →
  вертикаль); последняя вершина на правом крае.
- **Единый масштаб Y:** макс цена (среди buy/sell) → верх (`y == pad`), мин → низ (`y == height−pad`);
  подписи Y — макс и мин цены.
- **Вырожденные:** одна точка → `single`, плоская линия во всю ширину без деления на ноль по X;
  неизменная цена на разных датах → паддинг, линия по центру, одна подпись Y; пустой ряд →
  `has_data == false`, пустые линии.
- **HTTP:** `GET /products/{id}/prices` → 200 с `<svg>`, легендой и подписями; несуществующий id → 404;
  товар с одной точкой → «Цена не менялась».

**Живая проверка (TestClient на реальном ASGI-приложении):** создан товар → правка цены (закупка
50.00→60.00, продажа 100.00→130.00) → фрагмент `/products/{id}/prices` вернул **200 с двумя
`polyline`** (две кривые) и **без** пометки «Цена не менялась» (две точки → ступень построена).
Геометрия трёхточечного ряда проверена вручную: ступенчатые `buy_line`/`sell_line`, единый масштаб
(продажа 130.00 → верх, закупка 50.00 → низ), подписи дат/цен на местах. Временный товар удалён,
счётчик кодов возвращён к 0 — боевая `data/store.db` осталась чистой.

**Не покрыто автотестами:** фактическое исполнение `hx-trigger="load"` браузером (TestClient JS не
исполняет). Паттерн идентичен истории движений (3.2); проверен рендер самого фрагмента.

## 8. Технический долг / отложено

- **Интерактивность графика** (зум, tooltip, выбор диапазона) — не требуется макетом; статичный SVG.
  Улучшение при реальной необходимости.
- **Текущая цена у правого края.** Ось X по времени → новейшая точка ложится ровно на правый край,
  без видимой горизонтальной «полки до сих пор». Согласуется с формулой X из ТЗ; при желании показать
  полку до «сейчас» — добавить синтетический конечный узел на текущий момент (не блокер).
- **Экспорт/печать графика** — вне scope (этап 7.x).
- **Миграции** — как и в 3.1/3.2, схема через `create_all` без Alembic; новых полей нет, долг прежний.
